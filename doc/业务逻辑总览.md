# 业务逻辑总览

## 1. 项目概览

### 1.1 核心业务目标
IPAP Workflow 是一个专注于图像处理与自动化工作流的项目，主要功能包括：
- 项目结构自动创建与管理
- 图像处理与高清化
- 翻译文件与词汇表管理
- 工作流状态跟踪

### 1.2 技术栈说明
- **Python 3.12**：使用了现代 Python 语法特性，如类型标注、上下文管理器等
- **核心依赖**：
  - opencv-python：图像处理
  - numpy：数值计算
  - realcugan-ncnn-py：图像高清化
  - pillow：图像处理
  - pydantic：数据验证
  - tomlkit：配置文件解析
  - loguru：日志管理

### 1.3 架构分层与模块划分逻辑
- **入口层**：main.py 作为程序入口，负责启动工作流服务
- **服务层**：service.py 封装业务逻辑，提供工作流服务
- **核心层**：core.py 实现核心业务逻辑，包括项目创建、图像处理等
- **配置层**：config.py 负责配置管理与验证
- **工具层**：utils.py 提供通用工具函数，如图像处理、文件操作等
- **异常层**：exceptions.py 定义自定义异常类

## 2. 核心业务模块

### 2.1 src/service.py
- **职责**：工作流服务类，封装业务逻辑
- **对外接口**：
  - `load_settings(config_path)`：加载配置文件
  - `execute_workflow()`：执行工作流
- **依赖关系**：依赖 config.py 和 core.py

### 2.2 src/core.py
- **职责**：实现核心业务逻辑，包括项目创建、图像处理等
- **对外接口**：
  - `run_workflow(settings)`：执行主要工作流
  - `create_project_structure(base_dir, project_name)`：创建项目结构
  - `process_upscaling_tasks(image_paths, target_preprocessing_dir, upscale_ratio, model_select, num_workers)`：处理图片高清化任务
- **依赖关系**：依赖 config.py 和 utils.py

### 2.3 src/config.py
- **职责**：配置管理与验证
- **对外接口**：
  - `load_config(file_path)`：加载配置文件
  - `validate_config_file(file_path)`：验证配置文件
  - `parse_config_file(file_path)`：解析配置文件
- **依赖关系**：依赖 exceptions.py

### 2.4 src/utils.py
- **职责**：提供通用工具函数，如图像处理、文件操作等
- **对外接口**：
  - `natural_sort_key(s)`：生成自然排序键
  - `upscale_single_image(image_path, output_dir, upscale_ratio, model_selected)`：高清化单个图片
  - `copy_image_files_optimized(source_dir, target_dir)`：复制图片文件
  - `rename_and_copy_images(src_path, target_dir, start_index)`：重命名并复制图片
- **依赖关系**：依赖 realcugan-ncnn-py、opencv-python、numpy

### 2.5 src/exceptions.py
- **职责**：定义自定义异常类
- **对外接口**：
  - `IPAPWorkflowError`：所有自定义异常的基类
  - `ConfigurationError`：配置加载或解析失败时抛出
  - `SystemCheckError`：启动前环境或依赖校验失败时抛出
  - `ExternalToolError`：调用外部工具失败时抛出
  - `ImageWriteError`：图像写入失败的自定义异常
  - `MaxWorkersValueError`：max_workers 值错误

## 4. 核心业务规则与约束

### 4.1 项目结构规则
- 项目目录命名格式：{YYYY-MM-DD}_{project_name}
- 标准子目录结构：
  - 02_Preprocessing/raw_source
  - 02_Preprocessing/original_non_text_raw
  - 02_Preprocessing/inpainted
  - 02_Preprocessing/mask
  - 03_Translation
  - 04_Typesetting/workfiles
  - 04_Typesetting/final_pages

### 4.2 图像处理规则
- 支持的图片格式：.jpg, .jpeg, .png, .gif, .bmp, .webp
- 高清化处理条件：平均文件大小 < 1000 KB
- 高清化倍数：默认 2 倍，范围 1-4 倍
- 噪声检测：通过 DCT 系数分析判断图片压缩程度，决定是否启用降噪

### 4.3 配置规则
- 配置文件格式：TOML
- 配置验证：使用 Pydantic 进行数据验证
- max_workers 必须为正数
- upscale_timeout_sec 必须在 0-3600 秒之间

### 4.4 异常处理策略
- 使用自定义异常类继承自 IPAPWorkflowError
- 捕获并记录异常，确保程序不会因错误而崩溃
- 对磁盘空间不足等特殊错误进行专门处理
